Module statefile

source code

Python Support for reading state files that have been generated by Serif.

This module does not (yet) support reading binary files.

The following classes are used to encode the data read from the state files. The indentation indicates which objects contain which other types of objects. Note that some objects (namely, EntitySet, ValueMentionSet, and RelMentionSet) are used in two different contexts -- these objects are marked with [*], and listed twice.

In general, these data classes define attributes corresponding to all of the raw data that is stored in each corresponding Serif class. However, they may not define methods that match the methods in the corresponding Serif class.

I.e., these classes allow you to get at all the data that is stored in the C++ Serif classes, but no attempt was made to replicate the API of those classes.

Several objects that are represented using custom classes in C++ serif are encoded using raw values in the Python version. For example, Symbol values are just stored as strings.

load_doc_theory(source, stage=None, verbose=0, skip_sentence_breaks=True)

source code 

Load a document theory from a given source. This function is analagous to the C++ method ``loadDocTheory()`` in ``Serif/Generic/driver/DocumentDriver.h``.

Parameters:
  • source - The source that the document theory should be read from; this may be a filename, an open file-like object, or a `StateLoader`.
  • stage - The name of the stage that's being loaded. If specified, then this will be checked against the state file, to ensure that it mtaches.
  • verbose - If nonzero, display messages showing the progress of the state loader. Higher levels give more verbosity.
  • skip_sentence_breaks - If true, then expect the source to begin with a sentence breaks description; and skip over this description.
