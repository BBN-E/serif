Configuring the Serif Server

The server configuration is defined using a "parameter file". If you wish to customize the server's behavior, you can do so using a custom parameter file. The recommended way to create a custom parameter file is to create a new parameter file that begins by including the standard parameter file, and then overrides select parameters. The following example demonstrates what this looks like: 

Custom SERIF Parameter File
# Custom parameter file.
# Lines beginning with '#' are comments.

# Include the standard parameter file.
# (This line must come before any custom parameter overrides.)
INCLUDE /exports/serif/par/all.best-english.par

# Define custom parameter values.
OVERRIDE start_stage: START
OVERRIDE end_stage: parse
OVERRIDE experiment_dir: /var/serif/server_dir

Some of the parameters you might want to override include:

experiment_dir Specifies a directory where the Serif HTTP server keeps working files, including session logs and output files for persistant sessions.
start_stage
end_stage		 The the range of annotation stages that this SERIF server supports. See the list of stages for more information.
max_parser_seconds A timeout parameter for the parser. If the parser has not found a parse for a single sentence before this number of seconds has elapsed, then it will give up and use a flat parse instead.
parser_lambda
parser_max_entries_per_cell		 These parameters control how aggressively Serif prunes when parsing. The two recommended settings are:
  • parser_lambda=-5; parser_max_entries_per_cell=10: These settings (which are used by default) generate high-quality parses.
  • parser_lambda=-1; parser_max_entries_per_cell=5: These settings can be used to increase Serifâ€™s throughput by about 6x. Parse quality will be slightly lower.
