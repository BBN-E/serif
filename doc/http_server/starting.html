Starting the Serif HTTP Server

Command-Line Usage
Usage: Serif --http-server [OPTIONS] PARFILE

    PARFILE: The SERIF parameter file, specifying where models are located.

Options:
  --http-server            Run Serif in HTTP server mode
  --http-server-port N     The port number that the HTTP server should use to 
                             listen for incoming connections.
  --http-server-logdir dir If specified, then log all HTTP POST requests and 
                             their responses to this directory.
  -h [ --help ]            Describe the allowed options
  -v [ --verbose ]         Generate verbose output
  -q [ --quiet ]           Do not write messages to stdout/stderr
  -o dir                   The experiment directory.  This directory is used to 
                             store session logs, and may be used to cache output 
                             files.
  -p param=value           Override a specific parameter's value
  --subprocess-server      Run Serif in subprocess server mode.

There is a separate version of the HTTP Server for each language that can be processed by SERIF. These binaries are named SerifHTTPServer_Langauge, and are located in the bin directory of your Serif installation.

The only required argument is PARFILE, the absolute path of the Serif parameter file. This file lets Serif know where to find its trained model files, and sets a wide variety of parameters used to control those models (see the server configuration documentation for more details). Typically you will also want to specify a port number that Serif should listen on: 

Example
% setenv SERIF_HOME /exports/serif
% ${SERIF_HOME}/bin/x86_64/Serif ${SERIF_HOME}/par/english.par --http-server --http-server-port 8080

To verify that the server is running correctly, connect to it with your web browser. For example, if the server is running on the machine "serif.bbn.com", on port 8080, then you would type the following URL into your browser: "http://serif.bbn.com:8080/". If the server is running correctly, you will see a webpage containing the Serif HTTP Server documentation.
