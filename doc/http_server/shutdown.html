Shutting Down & Restarting the Serif Server

The SerifHTTP server defines four POST commands that can be used to shut down or restart the server:

POST Shutdown Shut down the server. The server will stop accepting connections, but will finish processing any client connections that are currently active.
POST ImmediateShutdown Shut down the server immediately. The server will stop accepting connections, and will close any client connections that are currently active.
POST Restart Restart the server. The server will stop accepting connections, and finish processing any client connections that are currently active. Once all active connections have been processed, the server will restart (using the same command-line arguments that it was originally run with). If the configuration file has changed, then it will be reloaded.
POST ImmediateRestart Restart the server immediately The server will stop accepting connections, and close any client connections that are currently active. It will then restart (using the same command-line arguments that it was originally run with). If the configuration file has changed, then it will be reloaded.

The following example illustrates the use of these commands. (Note that, per the HTTP specification, the request message must be terminated by a blank line.) 

Client Request
POST Shutdown HTTP/1.0
Content-Length: 0


Server Response
HTTP/1.0 200 OK
Content-Length: 133
Content-Type: text/html

<html>
<head>
<title>Serif HTTP Server: Shutdown</title>
</head>
<body>
Shutting down the Serif HTTP Server...
</body>
</html>

If you are accessing this webpage via a live Serif HTTP server, then you can shut down or restart that server from the Serif HTTP Server Home Page.
